Main revision goals:

1. Keep V2’s level-based day progression.
The viewer should click through the day before reaching the archive. Structure it like:
Title Screen → Morning Level → Class Level → Work Level → Evening/Overload Level → Archive Dashboard → Device Archive Pages / Sync Summary / Final Quote

2. Add more interactions before the archive.
I want at least 20 interactive moments before the viewer reaches the Archive Dashboard. These should feel like ordinary device events, not like a game or quiz. Each event can appear as a notification, screenshot, short video placeholder, reminder, message, watch buzz, tab, or device prompt.

Use small system-style buttons such as:
Open / Ignore / Snooze / Save / Dismiss / Reply / Continue / Check / Sync

3. Morning Level interactions.
Use iPhone and Apple Watch first. Add interactions like:
- iPhone alarm: Snooze / Wake Up
- Apple Watch buzz: Check / Ignore
- Calendar notification: Open / Dismiss
- Message notification: Reply / Ignore
- Camera roll memory: Save / Scroll Past

The morning should feel calm, quiet, and minimal.

4. Class Level interactions.
Activate the iPad. Add interactions like:
- iPad notes: Continue Notes / Switch App
- PDF reading: Highlight / Skip
- Canvas reminder: Open Assignment / Remind Me Later
- Apple Watch stand reminder: Stand / Dismiss
- iPhone notification interrupts notes: Check Phone / Stay Focused
- Screenshot prompt: Screenshot / Let It Pass

This level should still feel controlled, but interruptions should start building.

5. Work Level interactions.
Activate the MacBook. Add interactions like:
- Canvas page: Work on Assignment / Open New Tab
- Email notification: Open / Ignore
- Browser tabs multiply: Close Tabs / Keep Open
- iPad note syncs to MacBook: Accept Sync / Not Now
- Calendar deadline: Start Task / Snooze Reminder
- Social/video tab: Open / Resist

This level should feel busier and more fragmented.

6. Evening / Overload Level interactions.
All four devices should be active. Add interactions like:
- Low battery warning: Charge / Ignore
- Screen Time warning: View Report / Dismiss
- Multiple notifications across devices: Clear All / Open One / Ignore
- Apple Watch body summary: View / Dismiss
- Deadline reminder repeats: Work / Snooze Again
- Final sync prompt: Sync Day / Keep Going

This level should feel dense, layered, and overwhelming, but still intentionally designed.

7. Make choices feel like they leave traces.
When the viewer chooses Open, Ignore, Save, Snooze, etc., show a small system message like:
- Trace saved.
- Reminder ignored.
- Attention redirected.
- Deadline retained.
- Body recorded. Feeling unavailable.
- Emotional context unavailable.
- Repeated access detected.

These traces should make the final archive feel like it was built from the viewer’s interactions.

8. Keep the V3 Archive Dashboard.
At the end of the day, send the viewer to the Archive Dashboard. The Archive Dashboard should include four clickable device cards:
- iPhone: Memory / Attention
- Apple Watch: Body / Time
- iPad: Notes / School
- MacBook: Work / Tabs

Each device archive page should include placeholders for real screenshots, screen recordings, device data, notifications, notes, watch data, tabs, and other traces.

9. Keep View Sync Summary.
On the Archive Dashboard, include a clear button:
“View Sync Summary”

The Sync Summary should show a fake daily report:
- 4 devices active
- Notifications opened: __
- Notifications ignored: __
- Screenshots saved: __
- Tabs left open: __
- Notes edited: __
- Reminders snoozed: __
- Steps recorded: __
- Emotional context: not found

Then include a button:
“Continue”

This should lead to the final quote screen.

10. Keep the final quote screen.
The final screen should be mostly black, quiet, and minimal. Center this text:

“Everything was synced.
The feeling was not.”

Add smaller text underneath:
“1 day archived. Emotional context: not found.”

Add a button:
“Return to Archive”

This button should send the viewer back to the Archive Dashboard.

11. Add Restart Day button.
On the main Archive Dashboard, add a subtle button:
“Restart Day”

This should take the viewer back to the Title Screen or Morning Level, so the viewer can replay the experience.

12. Visual style.
Keep the style dark, minimal, elegant, Apple-inspired, and slightly melancholic. Use black, dark gray, white, muted blue-gray, rounded cards, thin borders, clean typography, and subtle notification-like design.

The beginning should feel calm and spacious. The middle should feel busier. The overload level should feel crowded and fragmented. The archive should feel organized but a little unsettling, like the system is trying to summarize a person through device traces.

Do not make the prototype feel like a colorful game. It should feel like an interactive digital archive / day simulation built from ordinary device decisions.
